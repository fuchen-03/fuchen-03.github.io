hexo使用入门

fuchen2025-12-222025-12-22

我来记录下自己是如何使用hexo搭建博客的

环境配置

主机：Windows 11

Node.js

直接去官网下载Node.js — 在任何地方运行 JavaScript

终端执行node -v与npm -v，显示版本号表示安装成功

Git

Git - Install，Windows需要勾选「Add Git to PATH」，不然在cmd中输入git，终端找不到，没有添加的话可以手动配置，Win+I按键->系统->系统信息->高级系统设置->环境变量->Path（一定是首字母大写Path）

hexo

终端执行npm install -g hexo-cli，输入hexo -v显示版本号表示成功

创建博客

创建一个文件夹，例如my_blog

进入my_blog，打开终端，建议打开git bash

输入命令hexo init生成框架，如下所示

输入npm install安装依赖

启动本地服务

hexo clean\hexo cl清除缓存文件 (db.json) 和已生成的静态文件 (public)。

hexo generate\hexo g生成静态文件。

hexo server\hexo s启动本地服务

将博客部署到github page上是一个比较好的选择，不用买节点域名

输入以下命令创建SSH密钥

添加密钥

进入[C:\Users\用户名\.ssh]目录（要勾选显示“隐藏的项目”），用记事本打开公钥id_rsa.pub文件并复制里面的内容。

进入Github页面，进入Settings页面，选择左边栏的SSH and GPG keys，点击New SSH key。Title随便取个名字，粘贴复制的id_rsa.pub内容到Key中，点击Add SSH key完成添加。

在打开Git bash，输入ssh -T git@github.com，一路回车，最终显示“Hi xxx! You've successfully……”即连接成功。

写第一篇文章

使用命令hexo new post "文章标题"，hexo就会根据框架中scaffolds/post.md作为模板文件进行一个创建，例如hexo new post "hexo使用入门"，会得到如下内容

可以根据自己需要修改模板文件，因为我的文章不多，就只设置了categories，在categories填写你创建的类别，博客就会根据进行一个分类

完善博客的操作

起初的框架中，并没有categories、tags、about、link等页面，这个必须自己配置

使用hexo new page "名字"就会在source文件夹下创建对应名字的文件夹

如果你想在categories下面继续创建子文件夹，就使用命令hexo new page --path source/categories/读书笔记 "读书笔记"，就会在categories文件夹下创建一个读书笔记文件夹，并生成一个读书笔记.md，然后在刚才创建的第一篇文章中的categories填入读书笔记就实现了分类
